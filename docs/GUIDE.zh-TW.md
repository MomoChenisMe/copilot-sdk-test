# CodeForge 使用者指南

本指南涵蓋 CodeForge 的所有功能與操作方式。CodeForge 是一款部署在 Linux VPS 上的 AI 輔助開發工具，透過瀏覽器即可進行完整的程式開發工作。

---

## 快速入門

### 前置需求

在開始使用 CodeForge 之前，請確認已具備以下條件：

- **GitHub Copilot 訂閱** -- 需要有效的 GitHub Copilot 訂閱（Individual、Business 或 Enterprise 方案皆可）。CodeForge 透過 GitHub Copilot SDK 驅動 AI 功能，因此訂閱是必要條件。
- **Node.js v24+** -- 伺服器端需安裝 Node.js v24 或以上版本。請使用 `node --version` 確認版本。
- **Linux VPS** -- CodeForge 設計為部署在 Linux VPS 上運行。建議搭配 Nginx 作為反向代理，並使用 Let's Encrypt 取得 SSL 憑證，以確保 HTTPS/WSS 連線安全。

### 登入

CodeForge 採用密碼認證機制，啟動伺服器時須設定 `WEB_PASSWORD` 環境變數。

**密碼認證：**

1. 在瀏覽器中開啟 CodeForge 的網址。
2. 系統會顯示登入頁面，輸入密碼（至少 8 個字元）。
3. 點擊「登入」按鈕。
4. 認證成功後，伺服器會發放 session cookie，後續請求自動帶入，無需重複登入。

**Session Cookie：**

- 登入成功後，系統會在瀏覽器中設定 session cookie。
- 只要 cookie 有效，重新開啟頁面時會自動恢復登入狀態。
- 如需登出，可在 Settings > General 中點擊「登出」。

**密碼重設：**

若忘記密碼，可透過伺服器端的 reset-password CLI 指令取得一次性 Token，在登入頁面點擊「忘記密碼？」進入重設流程，輸入 Token 與新密碼即可完成重設。

**安全機制：**

- 登入嘗試過於頻繁時，系統會啟動速率限制（Rate Limiting）。
- 多次失敗後帳號會被暫時鎖定。

## 介面總覽

CodeForge 的介面由四個主要區域組成：頂部欄、聊天區域、輸入區域和側邊面板。

### 頂部欄

頂部欄位於畫面最上方，提供導航和全域操作功能：

- **頁簽列** -- 顯示目前開啟的所有頁簽，支援三種類型：Copilot、Terminal、Cron。點擊頁簽可快速切換，支援拖拉排序。
- **新增頁簽按鈕** -- 點擊「+」按鈕可新增頁簽。
- **歷史記錄按鈕** -- 開啟對話歷史記錄側邊面板，可搜尋和瀏覽過往對話。
- **設定齒輪按鈕** -- 開啟設定頁面，管理各項系統設定。
- **主題切換按鈕** -- 在深色主題和淺色主題之間切換（基於 Slate-Indigo 設計系統）。
- **連線狀態徽章** -- 顯示 WebSocket 連線狀態：「已連線」、「連線中」或「已斷線」。

### 聊天區域

聊天區域是與 AI 互動的主要空間：

- **訊息顯示** -- 使用者訊息和 AI 回覆以對話氣泡形式呈現。
- **串流回應** -- AI 回覆會即時串流輸出，降低等待感。
- **工具呼叫結果** -- 當 AI 執行工具（如讀檔、執行指令）時，會顯示工具名稱、參數和執行結果。
- **推理區塊** -- 顯示 AI 的思考過程（Claude 風格），包含思考耗時。

### 輸入區域

輸入區域位於畫面底部：

- **訊息輸入框** -- 輸入要傳送給 AI 的訊息。
- **檔案附件按鈕** -- 點擊迴紋針圖示可上傳檔案作為對話附件（上限 10MB）。
- **斜線指令（/）** -- 在輸入框中鍵入 `/` 可呼叫斜線指令選單，包括系統命令（如 `/clear`、`/settings`、`/new`、`/context`、`/cron`）和已啟用的技能。
- **@ 檔案引用** -- 在輸入框中鍵入 `@` 可引用專案中的檔案，將檔案內容作為上下文附加到訊息中。
- **輸入歷史** -- 使用鍵盤上下箭頭鍵可瀏覽先前傳送過的訊息。

### 側邊面板

側邊面板位於畫面右側，可展開或收合：

- **Artifacts 面板** -- 顯示 AI 生成的 Artifacts，支援多種類型的內容渲染。
- **OpenSpec 面板** -- 顯示 OpenSpec 規格驅動開發的儀表板，管理 changes、specs 和任務。

## Copilot Agent

Copilot Agent 是 CodeForge 的核心功能，透過 GitHub Copilot SDK 驅動，提供完整的 AI 程式設計代理能力。

### 發送訊息與接收串流回應

在輸入框中鍵入訊息，按下 Enter 或點擊發送按鈕即可向 AI 傳送訊息。AI 會以串流方式即時輸出回覆，無需等到完整回覆生成完畢。串流過程中，可點擊「停止」按鈕中斷回應。

### 模型切換

CodeForge 支援在對話中切換不同的 AI 模型：

- 點擊輸入區域的模型選擇器，或使用快捷鍵開啟模型選擇面板。
- 可用模型列表從 GitHub Copilot API 動態取得。
- 每個模型旁會標示 Premium 倍率（Premium Request 消耗量），協助你評估使用成本。
- 切換模型後，後續訊息將使用新選定的模型回覆。

### 工具呼叫

AI 可以使用多種工具來協助開發工作：

- **檔案讀取** -- 讀取專案中的檔案內容。
- **檔案寫入** -- 建立新檔案或修改現有檔案。
- **指令執行** -- 在伺服器上執行 Shell 指令。
- **程式碼搜尋** -- 在專案中搜尋特定的程式碼片段或模式。
- **Git 操作** -- 執行 Git 相關操作，如 commit、diff、log 等。

工具呼叫時，介面會顯示工具名稱和傳入的參數，執行完成後會展示結果。你可以展開或收合工具呼叫的詳細資訊。

### 推理區塊

當 AI 模型支援推理模式（如 Claude 的 extended thinking）時，回覆中會包含推理區塊。推理區塊以可展開的區段顯示，標題會顯示思考耗時（例如「思考了 12 秒」）。展開後可看到 AI 的完整思考過程，有助於理解 AI 的決策邏輯。

### Plan & Act 模式

Plan & Act 模式讓你可以在規劃和執行之間切換：

- **Plan 模式（計畫）** -- AI 僅進行規劃，不會實際執行任何工具。適合用於評估方案、討論策略，避免 AI 直接修改程式碼。
- **Act 模式（執行）** -- AI 會根據對話內容實際執行工具操作。這是預設模式。

透過輸入區域旁的 Plan/Act 切換按鈕或快捷鍵切換。在 Plan 模式下完成規劃後，系統會提示是否切換到 Act 模式開始執行。

### 網路搜尋

CodeForge 整合了 Brave Search API，讓 AI 可以搜尋網路資訊：

- 網路搜尋功能可逐對話開啟或關閉。
- 支援兩種模式：「自動」（AI 自行判斷是否需要搜尋）和「始終」（每次回覆都強制搜尋）。
- 使用前須在 Settings > WebSearch 中設定 Brave Search API Key。

## Terminal

Terminal 提供真實的終端模擬器，而非簡易的指令執行器。

### 真實 PTY 終端

Terminal 基於 node-pty 和 xterm.js 實作，提供完整的 PTY（Pseudo Terminal）體驗：

- 完整的 ANSI 色彩與控制序列支援。
- 支援終端機調整大小（Resize），自動適應視窗寬度。
- 與原生終端機行為一致。

### 支援互動式程式

由於採用完整的 PTY 實作，Terminal 可以運行互動式程式：

- **vim / nano** -- 文字編輯器。
- **htop / top** -- 系統監控工具。
- **less / more** -- 分頁檢視工具。
- 其他任何需要 TTY 的互動式程式。

### 工作目錄管理

透過 CwdSelector（工作目錄選擇器）管理 Terminal 的工作目錄：

- 點擊工作目錄路徑可開啟目錄選擇器。
- 支援目錄搜尋、上層目錄導航。
- 選擇新目錄後，Terminal 會切換到該目錄。
- 也可直接輸入路徑快速跳轉。

### AI 與 Terminal 模式切換

在 Terminal 頁簽中，可以在 AI 模式和 Bash 模式之間切換：

- **AI 模式** -- 輸入的內容會傳送給 AI 處理，AI 可以代為執行指令。
- **Bash 模式** -- 輸入的內容直接作為 Shell 指令執行，等同於一般終端機。

使用快捷鍵或模式切換按鈕進行切換。

## 對話管理

CodeForge 提供完整的對話管理功能，所有對話記錄均以 SQLite 持久化保存。

### 建立和切換頁簽

- 點擊頂部欄的「+」按鈕新增頁簽。
- 點擊已存在的頁簽即可切換。
- 使用快捷鍵 Ctrl/Cmd + 1~9 快速切換到對應的頁簽。
- 使用快捷鍵切換到上一個或下一個頁簽。

### 頁簽類型

CodeForge 支援三種頁簽類型：

- **Copilot** -- AI 對話頁簽，用於與 Copilot Agent 互動。
- **Terminal** -- 終端機頁簽，提供完整的 PTY 終端。
- **Cron** -- 排程任務頁簽，管理和監控定時任務。

### 對話歷史記錄搜尋

點擊頂部欄的歷史記錄按鈕，開啟對話歷史側邊面板：

- 輸入關鍵字可在所有歷史對話中搜尋。
- 搜尋結果以時間排序呈現。
- 點擊搜尋結果可直接開啟該對話。

### 釘選重要對話

對於經常需要參考的重要對話，可以將其釘選：

- 在歷史記錄中，右鍵點擊或長按對話，選擇「釘選」。
- 已釘選的對話會顯示在列表頂部的「已釘選」區域。
- 再次操作可「取消釘選」。

### 刪除對話

- 在歷史記錄中，右鍵點擊或長按對話，選擇「刪除」。
- 系統會要求確認，確認後對話將被永久刪除。

### 頁簽拖拉排序

頁簽支援拖拉排序：

- 長按或拖動頁簽可調整其在頁簽列中的位置。
- 鬆開後頁簽順序會自動保存。

## 排程任務（Cron）

排程任務功能讓你可以設定定時自動執行的 AI 任務或 Shell 指令。

### 開啟排程設定面板

- 在 Cron 類型的頁簽中管理排程任務。
- 點擊「+ 新增任務」按鈕建立新的排程任務。
- 每個任務可獨立設定名稱、排程、類型和內容。

### 啟用/停用切換

每個排程任務都有獨立的啟用/停用開關：

- 開啟時，任務會按照設定的排程自動執行。
- 關閉時，任務暫停，但設定保留。

### 排程類型

支援三種排程類型：

- **Cron 表達式** -- 使用標準 Cron 表達式定義排程，例如 `*/5 * * * *` 表示每 5 分鐘執行一次。支援完整的五位 Cron 語法（分、時、日、月、週）。
- **時間間隔（毫秒）** -- 以固定的毫秒間隔重複執行，例如 `60000` 表示每 60 秒執行一次。
- **一次性（日期時間）** -- 在指定的日期時間執行一次。

### Cron 模型選擇

- 每個排程任務可獨立選擇 AI 模型。
- 任務類型支援 AI（AI 對話）和 Shell（Shell 指令）兩種。

### Prompt 設定

- 對於 AI 類型的排程任務，須設定 Prompt（提示詞）。
- 每次排程觸發時，系統會自動將該 Prompt 作為訊息傳送給 AI。
- 對於 Shell 類型的排程任務，須設定要執行的命令和工作目錄。

### 執行歷史

- 每個排程任務都會記錄執行歷史。
- 歷史紀錄包含開始時間、結束時間、持續時間、狀態（成功/錯誤/逾時）。
- 可查看每次執行的 AI 回應、工具呼叫和 Token 用量。

## Artifacts

Artifacts 是 AI 生成內容的獨立渲染面板，位於畫面右側。

### 檢視生成的 Artifacts

當 AI 在回覆中生成 Artifact 時，右側面板會自動展開並顯示內容。Artifacts 以獨立的渲染環境呈現，提供比聊天氣泡更好的閱讀和互動體驗。

### 支援類型

Artifacts 支援以下內容類型：

- **Markdown** -- 完整的 Markdown 渲染，支援標題、列表、表格、程式碼區塊等。
- **Code** -- 程式碼高亮顯示，支援多種程式語言。
- **HTML** -- 在安全隔離的沙盒 iframe 中渲染 HTML 內容，可執行 JavaScript。
- **SVG** -- 直接渲染 SVG 向量圖形。
- **Mermaid 圖表** -- 渲染 Mermaid 圖表語法，支援流程圖、序列圖、甘特圖等。

### 多 Artifact 頁簽切換

當一次對話中生成多個 Artifacts 時：

- 面板頂部會顯示頁簽列，每個 Artifact 各佔一個頁簽。
- 點擊頁簽可在不同 Artifacts 之間切換。

### 複製到剪貼簿與下載

每個 Artifact 都提供兩個操作按鈕：

- **複製** -- 將 Artifact 的原始內容複製到剪貼簿。
- **下載** -- 將 Artifact 下載為對應格式的檔案。

## OpenSpec（規格驅動開發）

OpenSpec 實作了規格驅動開發（Specification-Driven Development, SDD）流程，讓 AI 能夠基於明確的規格文件進行開發。

### 啟用 SDD

1. 開啟 Settings（設定）。
2. 切換到 OpenSpec 頁簽。
3. 開啟「啟用 OpenSpec SDD」開關。
4. 啟用後，每次 AI 對話都會注入 OpenSpec SDD 工作流程規則。

在此頁面中也可編輯 OPENSPEC_SDD.md 檔案，自訂 SDD 工作流程規則。

### 總覽儀表板

在 OpenSpec 面板的「總覽」頁簽中，可以看到專案的全局統計資訊：

- **進行中變更（Active Changes）** -- 目前正在進行的變更數量。
- **規格數量（Specs）** -- 專案中的規格文件總數。
- **已封存（Archived）** -- 已完成並封存的變更數量。

同時顯示專案設定，包括專案說明和產出規則。

### 管理 Changes

Changes 代表專案的一次性變更單元，可包含提案、設計和差異規格：

- **建立** -- 透過 AI 對話建立新的 Change，AI 會依照 SDD 流程產生結構化的變更文件。
- **檢視詳情** -- 在 Changes 列表中點擊某個 Change，可查看其詳細內容，包含任務列表、提案、設計和差異規格。
- **歸檔** -- 完成的 Change 可以歸檔，移至「已封存」區域。
- **刪除** -- 不需要的 Change 可以刪除（需二次確認）。

### 任務追蹤

每個 Change 都包含任務列表：

- 任務以 checkbox 形式呈現。
- 勾選 checkbox 可標記任務為已完成。
- 面板會顯示完成進度（例如「3/5 項任務」）。

### 檢視 Proposals、Designs、Delta Specs

每個 Change 的詳情頁面包含四個子頁簽：

- **任務（Tasks）** -- 該 Change 的任務列表和完成進度。
- **提案（Proposal）** -- 變更提案文件，描述變更的動機和範圍。
- **設計（Design）** -- 設計文件，描述技術方案和實作細節。
- **差異規格（Delta Specs）** -- 此次變更所涉及的規格差異。

### 瀏覽主要 Specs

在「規格」頁簽中，可以瀏覽專案的所有主要規格文件。每個規格文件以 Markdown 格式渲染，提供完整的閱讀體驗。

## 設定

設定頁面提供完整的系統設定管理，分為多個區域。

### 一般（General）

- **語言切換** -- 在 English 和繁體中文之間切換介面語言，切換即時生效。
- **Copilot SDK 版本管理** -- 顯示目前的 SDK 版本，可檢查更新、檢視更新日誌，一鍵更新至最新版本。
- **推播通知** -- 啟用或停用瀏覽器推播通知（詳見「推播通知」章節）。
- **登出** -- 點擊登出按鈕可清除 session 並返回登入頁面。

### System Prompt（系統提示詞）

- 自訂 AI 的系統指示，控制 AI 的行為和回應風格。
- 以純文字編輯器呈現，可自由編寫指示內容。
- 提供「重置為預設」按鈕，可還原為預設的系統提示詞（重置前會要求確認）。

### Profile（個人檔案）

- 填寫使用者個人資訊，這些資訊會作為上下文提供給 AI。
- AI 可以根據你的個人資訊提供更精準的回覆。

### OpenSpec

- **SDD 開關** -- 啟用或停用 OpenSpec 規格驅動開發工作流程。
- **OPENSPEC_SDD.md 編輯器** -- 直接編輯 SDD 工作流程規則文件，自訂開發流程。

### Memory（記憶）

Memory 系統讓 AI 能夠記住和運用對話中的重要資訊：

- **Auto Memory 編輯器** -- 直接檢視和編輯 AI 的記憶內容，包含偏好設定、專案資訊和解決方案三個分類。
- **自動萃取開關** -- 開啟後，AI 會自動從對話中辨識並儲存重要資訊。
- **LLM Intelligence** -- 進階的記憶管理功能：
  - **品質閘門（LLM Gating）** -- 使用 LLM 過濾低品質的記憶事實，僅保留具體且持久的使用者偏好與專案資訊。
  - **智慧萃取（LLM Extraction）** -- 使用 LLM 從對話中提取隱含的偏好與上下文，取代正則表達式提取。
  - **記憶壓縮（LLM Compaction）** -- 定期使用 LLM 合併重複事實、移除過時資訊、整理記憶內容。
- **每功能模型選擇** -- 為品質閘門、智慧萃取和記憶壓縮各自選擇最適合的 AI 模型。
- **手動壓縮** -- 點擊「立即壓縮」按鈕可手動觸發記憶壓縮流程。

### Skills（技能）

技能系統擴充 AI 的能力，分為系統技能和使用者技能：

- **系統技能** -- 預設提供的內建技能（唯讀，無法修改或刪除）。標記為「系統」。
- **使用者技能** -- 自行建立或安裝的技能，可自由編輯和管理。
- **啟用/停用開關** -- 每個技能都有獨立的開關，可控制是否將其注入 AI 對話。
- **從 ZIP 安裝** -- 點擊「上傳技能包」，拖放 ZIP 檔案或點擊瀏覽選擇檔案。
- **從 URL 安裝** -- 點擊「從 URL 安裝」，輸入 GitHub URL 或直接連結。
- **AI 輔助建立** -- 點擊「AI 建立技能」，透過 AI 對話描述需求，AI 會自動產生技能檔案。

### API Keys

- **Web Search（Brave Search）** -- 設定 Brave Search API Key，啟用 AI 的網路搜尋能力。

### MCP（Model Context Protocol）

MCP 設定頁面管理 Model Context Protocol 伺服器：

- **新增伺服器** -- 點擊「新增伺服器」，設定名稱、傳輸方式（stdio 或其他）、指令和參數。
- **管理伺服器** -- 檢視已設定的 MCP 伺服器，查看連線狀態（已連線/已斷線）。
- **重啟/刪除** -- 對每個伺服器可執行重啟或刪除操作。
- **工具列表** -- 檢視每個 MCP 伺服器提供的工具清單。

## 推播通知

CodeForge 支援基於 PWA 的瀏覽器推播通知，讓你在長時間任務完成時即時收到通知。

### 啟用推播通知

1. 開啟 Settings（設定）。
2. 切換到 General（一般）頁簽。
3. 找到 Notifications（通知）區域。
4. 開啟「推播通知」開關。

### 權限處理

啟用推播通知時，瀏覽器會要求授予通知權限：

- 若瀏覽器跳出權限對話框，點擊「允許」即可。
- 若通知權限已被封鎖，系統會顯示提示訊息：「通知已被封鎖，請在瀏覽器設定中啟用。」此時需手動前往瀏覽器的網站設定頁面解除封鎖。

### 通知類型

推播通知支援以下事件類型：

- **Cron 任務完成** -- 排程任務（定時任務）執行完成時推播通知，讓你掌握自動化任務的執行狀況。
- **AI 回覆完成** -- 當 AI 回覆需要較長時間處理（例如複雜的工具呼叫鏈），完成時會推播通知，方便你在離開頁面後回來查看結果。

每種通知類型都有獨立的開關，可分別啟用或停用。

### 測試通知按鈕

在通知設定區域提供「發送測試通知」按鈕，點擊後會立即發送一則測試通知到瀏覽器，可用來確認通知功能是否正常運作。
